A Dialogue with a Master

诸葛越论 计算思维

诸葛老师，能否用一句话概括它的核心？

“把问题拆开来、找规律、抓本质、再写成可执行的步骤”

DECOMPOSITION

分解

“在兵法里讲‘分进合击’，在计算思维里又该怎么理解？”

同理。大军不可一拥而上，算法不能一次解决全部。把大问题切成若干子问题，每个子问题简单明晰，才有可能精准击破。例如要做一款出行 App，与其直接喊“我要做滴滴”，不如先拆：定位 → 车辆匹配 → 计价 → 支付。每一块都能独立开发、迭代、压测。

PATTERN RECOGNITION

模式

“分好了块，接下来是找‘模式’？”

对。人类能预见未来，靠的就是识别模式。刘备三请诸葛亮，我看的是他求贤若渴这条“模式”，推算出他会给我充分主导权。写程序更是如此：浏览器里 70% 的报错出在输入不合法，这就是模式。因此在接口层统一做校验，比到处 try-catch 高效得多。

ABSTRACTION

抽象

“有人说抽象最难。为什么？”

因为要“舍”。抽象是隔绝细节，只留下本质——就像经典象棋残局，只保留关键子力。设计一个“支付接口”，真正要暴露的也许只有金额、支付方式、回调 URL；至于加密、风控、记账，都藏在实现里，调用方无需也不该知道。

ALGORITHM

算法

“最后是算法，这一步是不是程序员专利？”

未必。算法只是“可重复执行的步骤”。诸葛连弩的装填、发射到冷却，本质也是算法。好的算法关注两点：正确性（不走火）和效率（射速）。现代工程里从排序到路径规划，无不是在二者之间权衡。

诸葛小结

计算思维像一把四刃剑

分解开锋

模式定向

抽象护身

算法制胜

拿它去破题，无论是软件系统还是人生难局，皆可“兵以正合，奇以制胜”。

“算法写不出，永远停留在抽象；
问题不拆开，永远被复杂吞没。”

受教了！
