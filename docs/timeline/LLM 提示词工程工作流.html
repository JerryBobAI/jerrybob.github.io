PROMPT ENGINEERING WORKFLOW

构建高效 LLM 提示词的
系统化工作流

通过一个清晰、迭代的四步流程，从定义目标到最终测试，系统性地打造、评估和优化高质量的大语言模型提示词。

01

定义目标与收集样本

首先，清晰定义你希望 LLM 完成的具体任务。随后，准备 5-7 个高质量的 “输入-输出”对 作为正面示例。如果可能，加入 1-2 个 “负面样本” 来精确界定任务边界，帮助模型理解不应做什么。

ads_click
02

生成初始提示词 (一步到位)

在一个独立的对话中，向 LLM 发出一个综合指令，直接请求生成一个最佳提示词。指令应包含 角色设定、任务描述，并附上所有准备好的正、负面样本作为效果示例，明确输出格式和约束。

edit_note
03

专家评估与迭代

立即转换角色，让 LLM 扮演 提示词工程专家。要求它根据预设的评估标准（如清晰性、简洁性、鲁棒性等）严格审视刚生成的提示词，并直接输出 3 个基于评估的、更优的 替代版本。

rate_review
04

测试与定稿

从 3 个替代版本中选择最佳版本。使用全新的、甚至是刁钻的 “对抗性”输入 对其进行压力测试。根据测试结果进行最后的手动微调，直至提示词表现稳定、可靠，最终定稿。
