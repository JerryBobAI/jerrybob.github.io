什么是 AI 智能体？

DEFINING AI AGENTS

AI 智能体是结合了 大语言模型 (LLMs) 进行推理和决策，并利用 工具 (Tools) 与真实世界交互的系统。它们能够以有限的人工参与完成复杂任务。

智能体被赋予特定角色和不同程度的自主权以实现其最终目标。它们还配备了 记忆 (Memory)，使其能够从过去的经验中学习并随时间提升性能。 SYSTEMS COMBINING LLMS FOR REASONING WITH TOOLS FOR REAL-WORLD INTERACTION.

AI 智能体核心组件

LLM (大语言模型): 提供推理能力 (规划、反思)。
工具 (Tools): 与外部服务交互 (如API, 搜索)。
记忆 (Memory): 存储信息 (短期、长期)。
提示/指令 (Prompt/Instructions): 定义任务和角色。

什么是智能体工作流？

UNDERSTANDING AGENTIC WORKFLOWS

通常，一个 工作流 (Workflow) 是一系列为达成特定任务或目标而设计的关联步骤。 而一个 智能体工作流 (Agentic Workflow) 则是由一个或多个AI智能体 动态执行 的一系列关联步骤。智能体被用户授予权限，使其拥有一定程度的自主权来收集数据、执行任务并在真实世界中做出决策。

何为“智能体性”？

WHAT MAKES A WORKFLOW AGENTIC?

当一个或多个AI智能体引导和塑造任务进展时，AI工作流就具备了“智能体性”。其关键在于智能体的能力：

制定计划 MAKE A PLAN

通过任务分解，将复杂任务拆分为更小的子任务，并确定最佳执行路径。

执行动作 EXECUTE ACTIONS

使用预定义的工具集和权限来完成任务并执行其生成的计划。

反思迭代 REFLECT & ITERATE

评估每一步的结果，必要时调整计划，并循环直到结果满意为止。

智能体工作流核心模式

CORE PATTERNS IN AGENTIC WORKFLOWS

智能体工作流是为完成特定最终目标而采取的结构化步骤系列。智能体的行为模式在其能力中扮演关键角色，核心模式包括规划、工具使用、反思以及多智能体协作。

规划模式 PLANNING PATTERN

允许智能体自主地将更复杂的任务分解为一系列更小、更简单的任务（任务分解）。这能减轻LLM的认知负荷，改进推理，并最大限度地减少幻觉和其他不准确性。 AUTONOMOUS TASK DECOMPOSITION FOR COMPLEX PROBLEMS.

工具使用模式 TOOL USE PATTERN

扩展了智能体的能力，使其能够与外部资源和应用程序（如API、数据库、代码解释器）动态交互，而不仅仅是检索数据。检索增强生成 (RAG) 是该模式的一个例子。 DYNAMIC INTERACTION WITH EXTERNAL RESOURCES AND APPLICATIONS.

反思模式 REFLECTION PATTERN

智能体评估自身行动的结果、识别错误、并从经验中学习以优化未来步骤。这使得工作流能够自我纠正、适应不断变化的环境，并持续改进其性能。 SELF-CORRECTION, ADAPTATION, AND CONTINUOUS IMPROVEMENT.

多智能体协作模式 MULTI-AGENT COLLABORATION

涉及多个具有不同专长或角色的智能体协同工作。它们通过任务分配、信息共享和结果整合来共同解决单个智能体难以应对的复杂问题，实现1+1>2的效果。 SYNERGISTIC PROBLEM-SOLVING WITH SPECIALIZED AGENTS.

注意：上述为简化示意。实际模式可能更复杂，并包含更多步骤和反馈循环。
